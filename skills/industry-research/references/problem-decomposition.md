# Problem Decomposition

Use this file when the user asks a specific industry question or when the report needs an explicit issue tree.

## Problem Definition

Problem definition means identifying the true question, the granularity, and the required decision. In this skill, the most important granularity is industry scope plus the user's actual question.

## 5W2H Question Definition

Use these prompts when needed:

- What happened?
- What is unsatisfactory or uncertain?
- What does the user want to know or decide?
- Where does it happen?
- When is the relevant period?
- Who is affected?
- How will the answer be used?
- How much depth or evidence is needed?

## MECE Issue Tree Rules

- Put the mother question first.
- Break it into mutually exclusive, collectively exhaustive sub-questions where feasible.
- Use causal structure for why/how questions.
- Use spatial/value-chain structure for industry structure questions.
- Use category grouping for player or segment comparison.
- Use logical progression from facts to mechanisms to implications.
- Make every sub-question verifiable by facts, data, or clearly marked expert opinions.

## Output Requirements

For specific-question reports:

1. Answer the mother question first.
2. Show the issue tree.
3. Link each conclusion to evidence.
4. Mark unanswered or weakly evidenced sub-questions in the verification list.
