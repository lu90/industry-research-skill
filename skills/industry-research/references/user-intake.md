# User Intake

Use this file to classify the request and lock the research boundary before analysis.

## Request Classification

| Type | Trigger | Required output behavior |
|---|---|---|
| Industry overview | User names an industry without a specific question | Produce a full industry overview report. |
| Industry-specific question | User asks why, whether, how, opportunity, risk, future trend, or cause inside an industry | Answer the question first, then provide the industry overview base and evidence chain. |
| Company/product analysis | User names a company, product, project, or business line | Use the industry overview base and add target-position and micro-analysis modules. |

## Boundary Questions

Use `5W2H / 6W2H` to clarify:

- What: What industry, question, company, product, or business line is being studied?
- Where: World, country, region, city, or market?
- When: Current state, historical review, next 1 year, next 3 years, next 5 years, or long term?
- Who: Industry, company, product, customer group, or value-chain segment?
- Why: Investment, entrepreneurship, operations, product strategy, learning, interview, or internal decision?
- How deep: quick overview, structured research note, deep report, decision recommendation?
- Boundary: What is included and excluded?

## Industry Scope

Force the scope into one of these口径:

- Narrow: only core segment or core product/service.
- Medium: core segment plus key upstream/downstream links.
- Broad: full industrial ecosystem.

## Clarification Policy

Ask only high-impact boundary questions. If the user asks to proceed without clarification, write explicit assumptions and continue. For ambiguous industry names, list the assumed scope in the report boundary.
