# Institutional Report Benchmarks

Use this reference to make standard and deep reports feel like formal institutional research. Do not copy external report text, headings, charts, or tables. Use the sources below only as layout and evidence-presentation benchmarks.

## Copyright Boundary

- Store links and high-level style lessons only.
- Do not paste complete reports or long excerpts into the skill.
- Do not imitate a single institution's proprietary report structure.
- Use public reports to learn report rhythm: executive summary, key takeaways, exhibits, data notes, assumptions, methodology, and source discipline.
- If a source has an open license, still summarize style instead of copying body text.

## Style Rules To Apply

1. Start standard and deep reports with a report-front section: executive summary, key conclusions, key metrics, and exhibit list.
2. Use exhibits as argument anchors. In Markdown, an exhibit can be a Mermaid map, metric table, comparison table, evidence table, or source-quality table.
3. Make each major chapter start with a paragraph-level conclusion before supporting details.
4. Use tables for metrics, evidence chains, comparisons, and validation lists, not as substitutes for reasoning.
5. Separate historical facts, forecast assumptions, and analyst inferences.
6. Include methodology and source notes near the end of the report.
7. Include an appendix-style verification list for missing data, weak evidence, and next checks.
8. Keep the skill's own industry-research structure as the source of truth; benchmarks affect presentation depth, not analytical logic.

## Minimum Formal Report Front

For standard or deep company/product reports, include these before the main body:

1. Report summary: 2-4 short paragraphs covering conclusion, industry context, target position, and main risks.
2. Key conclusions: 4-6 bullets with claim, reason, and evidence pointer.
3. Core metrics overview: a compact table of market, growth, penetration, profitability, competition, and target-specific metrics.
4. Exhibit list or exhibit placeholders: at least 4 items for standard reports and at least 6 for deep reports.

## Minimum Formal Report Back

For standard or deep company/product reports, include these after the main body:

1. Methodology and data source note.
2. Evidence quality summary.
3. Key assumptions.
4. Appendix: follow-up verification checklist.
