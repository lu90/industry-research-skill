# Multi-Agent Pressure Test

Use this after the main analysis and before final recommendations.

## Default Mode

Default to multi-agent division if the environment supports subagents:

- Organizer: controls scope, reconciles disagreements, and summarizes risks.
- Industry expert: checks business logic and value-chain assumptions.
- Investment researcher: checks valuation, data, and investability logic.
- Policy researcher: checks policy, regulation, and macro assumptions.
- Operator/entrepreneur: checks business model, execution, and go-to-market realism.
- Intern perspective: asks unconventional questions and spots ignored angles.
- Devil's advocate: challenges the central conclusion.

If subagents are unavailable, explicitly state that the analysis uses single-agent simulated perspectives.

## Rules

- Pressure testing must not create new unsupported facts.
- Every challenge must refer to facts, opinions, or inferences already collected, or mark itself as a question for further verification.
- Output only useful disagreements, risks, and verification needs.
- Do not add theatrical dialogue; summarize in analyst style.
