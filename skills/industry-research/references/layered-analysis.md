# Layered Analysis

Use this reference to decide which analysis layers must appear in the report. Do not force the micro layer into pure industry questions.

## Layer Selector

| Request type | Required layers | Optional layers | Rule |
|---|---|---|---|
| Industry overview | Macro + meso | Micro only for representative cases or players | Do not analyze a company unless it helps explain the industry. |
| Industry-specific question | Macro + meso + issue tree | Micro only when the question names or depends on a company/product/player | Answer the question first, then show the industry base. |
| Company/product analysis | Macro + meso + micro | Capital-market layer only when asked | Industry context must come before target-specific analysis. |
| Capital-market question | Macro + meso + micro + capital market | Portfolio/BCG only when relevant | Use company/product analysis as the base and add capital-market modules. |

## Layer Definitions

### Macro Layer

Use for external context: policy, regulation, macro economy, consumption cycle, rates, inflation, exchange rates, trade, technology cycle, and broad capital-market risk appetite. Macro does not always need a full PEST essay; weight it by the user's question.

### Meso Layer

Use for industry structure: industry definition, boundary, market size, growth, penetration, value chain, competitive structure, lifecycle, prosperity indicators, valuation logic, and profit-pool distribution. This layer is mandatory for industry research.

### Micro Layer

Use only when the request includes a company, product, project, brand, business line, representative player, or startup idea. Cover business model, product/service position, customers, channels, financials, operating indicators, moat, SWOT, competitors, and portfolio analysis when appropriate.

### Capital-Market Layer

Use only when the user asks about stock price, valuation, market expectation, investability, market-cap repair, whether a listed company may rise/fall, or why performance diverges from fundamentals.

## Multi-Business Companies

For multi-business companies, do not collapse the meso layer into one generic industry. Split the relevant industry lines. For example, Xiaomi may require smartphone, AIoT/consumer electronics ecosystem, internet services, and smart EV layers depending on the question.

Each relevant business line should include at least:

1. Industry stage or lifecycle.
2. Competitive structure.
3. Key indicators or prosperity signals.
4. What the industry line means for the target company.

If a business line is not relevant to the user's question, state that it is excluded by scope instead of adding a shallow section.

## Output Rule

In standard and deep reports, state the selected layers in the research boundary or methodology note. If a layer is skipped, make the reason clear when it could otherwise be expected.
